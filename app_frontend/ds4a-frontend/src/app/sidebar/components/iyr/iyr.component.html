Descripción de los datos

Se usaron 3 archivos para entrenar los modelos: "BD_MinTic_2010_2020.xlsx", "Componentes_Dimensiones_2016_2019.xlsx" y "MDM_2016_2019.xlsx", junto con datos adicionales de la base de datos TerriData, que contiene datos de varios sitios oficiales.
Cada base de datos tenía tanto datos numéricos como datos categóricos.

Anális Exploratorio de los Datos

A continuación, presentamos un análisis exploratorio de datos a partir de los datos proporcionados por el MinTIC.En primer lugar, comenzamos por limpiar los conjuntos de datos, que abarcó la estandarización de los nombres de las columnas para todas las tablas, la eliminación de caracteres no alfanuméricos y la sustitución de espacios por subrayados. Muchos campos de la tabla Componentes_dimensiones eran coberturas, pero tenían medidas inconsistentes para distintos años (por ejemplo, medidas como porcentajes para algunos años y como proporciones para otros), por lo que también estandarizamos todas las medidas de cobertura como proporciones entre 0 y 1.

En cuanto a la base de datos TerriData, el primer paso para hacer más manejable esta gran base de datos fue dividirla entre las dimensiones que contenía. Así, trabajamos con 18 tablas separadas, una por dimensión. La limpieza de cada tabla consistió en comprobar si los datos estaban completos o no para cada variable. Se consideró que los datos de una variable estaban completos cuando cumplía con dos condiciones: i) tenía una terminación territorial, es decir, cada entidad territorial tenía una medida asociada para la variable, y ii) tenía una terminación temporal, es decir, para cada entidad territorial que había en mediciones mínimas para los años 2016, 2017 y 2018. Si no se cumplía una de las dos condiciones, filtramos la variable. También decidimos filtrar las variables que tenían mediciones más de una vez al año, ya que no podíamos estar seguros de cuál era la primera o la última medición del año y qué mediciones eran parciales para ese año.

Además, hubo algunas variables como el PIB (pib en la base de datos) que se midieron solo a nivel de departamento, pero no a nivel municipal, por lo que al final hicimos dos conjuntos de datos para trabajar: un conjunto de datos para el nivel municipal y otro para el nivel de departamento. También resumimos todas las inversiones del MinTIC, primero por municipio y segundo por departamento para incluirlo como variable independiente en nuestros modelos.

A nivel municipal obtuvimos en total 171 variables con 4404 observaciones, que comprenden el período 2016-2019 (1101 observaciones por año), mientras que obtuvimos 209 variables con 3303 observaciones que comprenden el período 2016-2018. No faltaron datos aparte de todo el año 2019 para 38 variables. A nivel departamental obtuvimos en total 169 variables con 132 observaciones, que comprenden el período 2016-2019 (33 observaciones por año), mientras que obtuvimos 242 variables con 99 observaciones para el período 2016-2018. Las variables con datos para los cuatro años no tenían datos faltantes, mientras que algunas variables para el período 2016-2018 tenían algunos datos faltantes (entre tres y seis observaciones para cada variable). Imputamos estos datos faltantes utilizando la media anual de la variable respectiva.

Un dato importante sobre nuestros datos, y específicamente sobre la variable de inversiones MinTIC es que, en el caso de los municipios, más del 60% de los datos estuvieron por debajo de COP $ 1e8 y más del 15% de los datos fue efectivamente igual a COP $ 0. , mientras que el valor máximo fue del orden de COP $ 7e10 (Figura 1), lo que lo hace muy sesgado a la derecha y, de hecho, muy difícil de usar como variable independiente para nuestro modelo. Es por eso que decidimos usar solo el nivel de departamento, ya que tiene menos valores de COP $ 0 y está menos sesgado (Figura 2).

Conclusiones y Recomendaciones

Los tres modelos aplicados fueron similares en su rendimiento en la predicción de valores del grupo de testeo.
El modelo con menor error fue el modelo de regresión XGB, mientras que el modelo con mayor error fue el modelo Random Forest. Entre las variables más importantes para explicar la variabilidad en los modelos se encuentran Población Rural, Densidad de Población y PIB, que no son muy sorprendentes ya que sabemos que la calidad de vida depende en gran medida de la economía y la demografía de la sociedad.

Sin embargo, después de modelar nuestras tres estrategias, no encontramos una relación estadísticamente significativa entre las inversiones MinTIC y el índice de calidad de vida, lo cual es decepcionante pero no inesperado, ya que la calidad de vida es un fenómeno complejo relacionado con múltiples fuentes de variabilidad.
También pensamos que es necesario tener una cobertura más temporal de los datos para captar mejor la relación matizada entre las inversiones en tecnologías de la información y las comunicaciones y la calidad de vida de una sociedad.Es necesario realizar un análisis más detallado y considerar diferentes enfoques de la calidad de vida para determinar las correlaciones deseadas.
